01

Positioning & lane.

How to talk about it

Full positioning statement

My role is not to tell Schaff how to run construction projects. My role is to help Ryan and Jessica use Claude and JobTread more consistently, safely, and efficiently. I'll observe how the work happens now, identify where AI can reduce repetitive drafting or organizing, and help build simple review loops that keep humans in control of estimates, permits, budgets, contracts, client commitments, and financial decisions.

Shorter version

I'm here to help you get more practical value from Claude without making the workflow more complicated. We'll start with what you already do, find the parts that are repetitive or hard to keep organized, and build small Claude workflows that support your judgment instead of replacing it.

Loop framingShow the real work → Claude helps draft, check, or organize → human reviews → JobTread gets cleaner → repeat with less friction. This is workflow design, not AI hype.
02

Ryan tutoring session agenda.

~60 min
0–5 min
Set expectations

"Today isn't about automating the company. It's about finding two or three places where Claude can save you time this week."

  • Where do you lose the most time right now?
  • What do you repeat every week?
  • What do you avoid because it takes too much setup?
5–15 min
Claude Desktop tour
  • Standard chat vs. structured work chat
  • Starting a clean conversation for a specific task
  • Pasting notes, emails, project context
  • Asking for a specific output format
  • Treating Claude's output as a draft, not truth
15–30 min
Teach one repeatable prompt pattern

"Claude is strongest when you give it the messy input and a clear definition of the finished output."

30–45 min
Cowork walkthrough

Frame Cowork as "Claude for tasks that need a little more persistence, structure, or follow-through." Live example: project notes → missing-information checklist + clean internal update for human review before JobTread entry.

45–55 min
Build Ryan's first reusable workflow — "Project Notes Cleaner"

Input: raw notes from a call, site visit, text thread, email, or meeting. Output: clean summary, decisions, open questions, missing JobTread fields, risks/unclear commitments, draft JobTread update, items needing Ryan/Jessica review.

55–60 min
Assign one practice task

"Before next time, take one messy note or client update, run it through the Project Notes Cleaner, and bring me the output — we'll improve the workflow."

03

First working-session agenda.

~90 min · "show me how this works now"
0–10 min
Frame the session

"I'm not here to judge the workflow or prescribe a construction process. I'm here to understand where information enters, where it gets stuck, and where Claude or JobTread could support the process."

10–25 min
Show me JobTread
  • Where a new lead enters; where notes live; where project status lives
  • Where tasks/follow-ups, estimate notes, permit status live
  • Where client communications are stored or referenced
  • Which fields people trust, ignore, or work around
25–40 min
Show me a real recent workflow

"Can you show me one recent project or lead where the workflow felt messy?" Watch for: duplicate entry, missing info, data living in texts/emails, unclear ownership, manual retyping, memory-dependent follow-ups.

40–55 min
Show me Claude usage

Ryan: show me the last thing you asked Claude — what did it do well, where didn't you trust it, what would make it more useful? Jessica: what already works, what would be annoying for me to re-explain, where would a shared template help?

55–70 min
Identify one safe pilot candidate

Look for: lead/project notes → Claude summary → missing-info checklist → JobTread-ready update → human review.

70–90 min
Confirm next step

"I'll write up a short findings note and recommend one low-risk pilot. If it looks useful, we test it for two weeks. If it doesn't, we stop there."

Help me prepare for a client follow-up.

Context: I need to respond professionally, but I want to make sure I do not overpromise.

Input: [paste client message or notes]

07

Discovery tools.

If the work expands

Needs-assessment survey

  • What recurring task takes more time than it should?
  • Where does information get lost or duplicated?
  • Which JobTread fields do you trust most? Which do you work around?
  • What have you used Claude for successfully? Where didn't you trust it?
  • Which workflows need human approval no matter what?

Claude usage audit

  • What prompts are already working?
  • Who uses Claude most effectively?
  • Where are outputs reviewed before use?
  • Where does hallucination/overconfidence create risk?
  • Which repeatable tasks deserve templates?

CRM trust audit

  • Source of truth for leads, projects, notes, estimates, permits, client comms, tasks, status?
  • Where does data enter JobTread? Where does it stay outside?
  • Which fields are incomplete, stale, duplicated, or ignored?
  • What would make Ryan/Jessica trust the system more?

Data-protection checklist

  • Redact sensitive client details when possible.
  • Don't paste unnecessary financial, contract, or private info.
  • Keep client-facing commitments under human review.
  • Document what's safe to use with Claude.
  • Use templates that instruct Claude not to invent facts.
  • Keep JobTread as the source of truth for official records.
08

One-page findings template.

Keep the first deliverable short
  1. What's already working — where Ryan, Jessica, Claude, or JobTread already help the work move.
  2. Where the team is losing time — repeated drafting, missing info, duplicate entry, unclear ownership, memory-dependent follow-ups.
  3. Where JobTread data/trust is weak — incomplete fields, stale status, info outside the CRM, areas people avoid.
  4. Where Claude can help safely — summarizing, drafting, checklists, missing-info detection, internal updates, with human review.
  5. Recommended next pilot — one low-risk loop with inputs, outputs, review gates, and success metrics.
